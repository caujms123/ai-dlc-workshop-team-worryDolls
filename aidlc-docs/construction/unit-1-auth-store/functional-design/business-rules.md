# Unit 1: 비즈니스 규칙 - 인증 + 매장 + 관리자 + 광고

---

## 1. 인증 규칙

### BR-AUTH-01: 비밀번호 정책
- 최소 8자 이상
- bcrypt 해싱 (cost factor: 12)
- 평문 비밀번호는 절대 저장/로깅하지 않음

### BR-AUTH-02: JWT 토큰 정책
- 토큰 유효 기간: 16시간
- 토큰 페이로드: `{ sub: admin_id/table_id, role: SUPER_ADMIN/STORE_ADMIN/TABLE, store_id: int, exp: timestamp }`
- 서명 알고리즘: HS256
- 토큰 갱신: 만료 전 재로그인 필요 (자동 갱신 없음)

### BR-AUTH-03: 로그인 시도 제한
- 연속 5회 실패 시 계정 잠금
- 잠금 시간: 15분
- 잠금 해제 후 시도 횟수 리셋
- 성공 로그인 시 시도 횟수 리셋

### BR-AUTH-04: 역할 기반 접근 제어 (RBAC)
| 역할 | 접근 범위 |
|---|---|
| SUPER_ADMIN | 모든 매장, 모든 관리 기능 |
| STORE_ADMIN | 소속 매장만, 매장 관리 기능 |
| TABLE | 소속 매장의 메뉴 조회, 주문 생성, 주문 내역 조회 |

### BR-AUTH-05: 매장 관리자 로그인 플로우
1. store_code로 매장 존재 여부 확인
2. 매장 활성 상태 확인
3. username으로 관리자 조회 (해당 매장 내)
4. 관리자 활성 상태 확인
5. 비밀번호 검증
6. 로그인 시도 제한 확인
7. JWT 토큰 발급

### BR-AUTH-06: 테이블 태블릿 로그인 플로우
1. store_code로 매장 존재 여부 확인
2. table_number로 테이블 조회 (해당 매장 내)
3. 비밀번호 검증
4. JWT 토큰 발급 (role: TABLE)
5. 토큰을 클라이언트 로컬 스토리지에 저장

### BR-AUTH-07: 자동 로그인 플로우
1. 로컬 스토리지에서 저장된 토큰 확인
2. 토큰 유효성 검증 (만료 여부)
3. 유효하면 자동 로그인 성공
4. 만료되었으면 저장된 자격 증명으로 재로그인 시도

## 2. 매장 규칙

### BR-STORE-01: 매장 식별자 규칙
- store_code는 시스템 전체에서 고유
- 영문 소문자 + 숫자 + 하이픈만 허용
- 3~30자 길이 제한
- 등록 후 변경 불가

### BR-STORE-02: 매장 등록 필수 필드
- store_code (필수, 고유)
- name (필수)
- address (선택)
- phone (선택)

### BR-STORE-03: 매장 비활성화 규칙
- 매장 비활성화 시 해당 매장의 관리자 로그인 차단
- 매장 비활성화 시 해당 매장의 테이블 로그인 차단
- 기존 데이터(메뉴, 주문 등)는 보존

---

## 3. 관리자 규칙

### BR-ADMIN-01: 관리자 생성 규칙
- 슈퍼 관리자만 관리자 계정 생성 가능
- 동일 매장 내 username 중복 불가
- 슈퍼 관리자의 store_id는 NULL
- 매장 관리자의 store_id는 필수

### BR-ADMIN-02: 관리자 비활성화 규칙
- 비활성화된 관리자는 로그인 불가
- 비활성화해도 기존 데이터는 보존
- 슈퍼 관리자는 자기 자신을 비활성화할 수 없음

### BR-ADMIN-03: 초기 슈퍼 관리자
- 시스템 초기 설정 시 기본 슈퍼 관리자 계정 1개 생성 (DB seed)
- username: admin, password: 초기 비밀번호 (첫 로그인 시 변경 권장)

## 4. 광고 규칙

### BR-AD-01: 광고 이미지 업로드 규칙
- 허용 형식: JPG, JPEG, PNG
- 최대 파일 크기: 5MB
- 파일명: UUID 기반 고유 파일명 생성
- 저장 경로: `uploads/advertisements/{store_id}/{uuid}.{ext}`

### BR-AD-02: 광고 노출 규칙
- 활성(is_active=true) 광고만 고객 화면에 표시
- display_order 오름차순으로 표시
- 5초 간격 자동 슬라이드
- 활성 광고가 없으면 기본 이미지 표시 또는 바로 메뉴 화면

### BR-AD-03: 광고 삭제 규칙
- 삭제 시 서버의 이미지 파일도 함께 삭제
- 삭제 후 나머지 광고의 display_order 재정렬

### BR-AD-04: 광고 순서 변경 규칙
- display_order 값을 직접 지정하여 순서 변경
- 동일 매장 내에서만 순서 관리

---

## 5. 파일 업로드 규칙

### BR-FILE-01: 이미지 검증
- Content-Type 검증 (image/jpeg, image/png)
- 파일 확장자 검증 (.jpg, .jpeg, .png)
- 파일 크기 제한: 5MB

### BR-FILE-02: 파일 저장
- 저장 디렉토리: `uploads/{category}/{store_id}/`
- 파일명: `{uuid}.{ext}` (원본 파일명 사용하지 않음)
- 반환값: 상대 경로 URL

### BR-FILE-03: 파일 삭제
- 파일 경로로 물리적 파일 삭제
- 파일이 존재하지 않아도 에러 발생하지 않음 (idempotent)
