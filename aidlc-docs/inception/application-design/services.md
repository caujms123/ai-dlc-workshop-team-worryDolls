## 2. 서비스 정의

### 2.1 AuthService
**책임**: 인증/인가 전체 오케스트레이션
**오케스트레이션 패턴**:
- 로그인 요청 → RateLimiter 확인 → 자격 증명 검증 → JWT 발급
- 토큰 검증 → 사용자 정보 추출 → 역할 확인

**의존 서비스**: StoreService (매장 검증), AdminRepository, TableRepository

---

### 2.2 StoreService
**책임**: 매장 비즈니스 로직
**오케스트레이션 패턴**:
- 매장 등록 → 식별자 고유성 검증 → 저장

**의존 서비스**: StoreRepository

---

### 2.3 AdminService
**책임**: 관리자 계정 비즈니스 로직
**오케스트레이션 패턴**:
- 관리자 생성 → 사용자명 중복 검증 → 비밀번호 해싱 → 저장

**의존 서비스**: AdminRepository, AuthService (비밀번호 해싱)

---

### 2.4 AdvertisementService
**책임**: 광고 이미지 비즈니스 로직
**오케스트레이션 패턴**:
- 광고 업로드 → 이미지 검증 → 파일 저장 → DB 저장
- 광고 삭제 → 파일 삭제 → DB 삭제

**의존 서비스**: FileUploadService, AdvertisementRepository

---

### 2.5 MenuService
**책임**: 메뉴 비즈니스 로직
**오케스트레이션 패턴**:
- 메뉴 등록 → 필수 필드 검증 → 가격 검증 → 이미지 업로드 → 저장
- 메뉴 삭제 → 이미지 파일 삭제 → DB 삭제

**의존 서비스**: FileUploadService, MenuRepository, CategoryRepository

---

### 2.6 OrderService
**책임**: 주문 비즈니스 로직 (핵심 서비스)
**오케스트레이션 패턴**:
- 주문 생성 → 세션 확인/시작 → 메뉴 유효성 검증 → 금액 계산 → 저장 → SSE 이벤트 발행
- 상태 변경 → 저장 → SSE 이벤트 발행 (관리자 + 고객)
- 주문 삭제 → 삭제 → 총 주문액 재계산 → SSE 이벤트 발행
- 이용 완료 → 주문 이력 이동 → 세션 종료 → SSE 이벤트 발행

**의존 서비스**: TableService, MenuService, SSEService, OrderRepository

---

### 2.7 TableService
**책임**: 테이블 및 세션 비즈니스 로직
**오케스트레이션 패턴**:
- 이용 완료 → OrderService.move_to_history → 세션 종료 → 리셋

**의존 서비스**: TableRepository, OrderService (이력 이동)

---

### 2.8 SSEService
**책임**: 실시간 이벤트 스트리밍 관리
**오케스트레이션 패턴**:
- 구독 등록 → 연결 유지 → 이벤트 수신 시 전달
- 이벤트 발행 → 해당 매장/테이블 구독자에게 전달

**의존 서비스**: 없음 (독립적, 다른 서비스에서 호출)

---

### 2.9 FileUploadService
**책임**: 파일 업로드 공통 로직
**오케스트레이션 패턴**:
- 파일 검증 → 고유 파일명 생성 → 디스크 저장 → URL 반환

**의존 서비스**: 없음 (독립적 유틸리티)

## 3. 미들웨어 정의

### 3.1 AuthMiddleware
- JWT 토큰 검증
- 사용자 정보를 요청 컨텍스트에 주입
- 역할 기반 접근 제어 (SUPER_ADMIN, STORE_ADMIN, TABLE)
- 공개 엔드포인트 제외 처리

### 3.2 RateLimiter
- 로그인 엔드포인트 시도 제한
- IP 기반 또는 식별자 기반 제한
- 제한 초과 시 429 응답

### 3.3 GlobalErrorHandler
- 전역 예외 처리
- 구조화된 에러 응답 (코드, 메시지)
- 프로덕션 환경에서 스택 트레이스 숨김
- 에러 로깅

## 4. 통신 패턴

### 4.1 동기 통신
- **Client → Server**: REST API (HTTP/JSON)
- **Service → Service**: 직접 메서드 호출 (모놀리식 내부)
- **Service → Repository**: SQLAlchemy ORM

### 4.2 비동기 통신
- **Server → Client**: SSE (Server-Sent Events)
  - 관리자: 매장별 주문 이벤트 스트림
  - 고객: 테이블별 주문 상태 스트림
