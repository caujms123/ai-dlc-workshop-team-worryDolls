# 테이블오더 서비스 - 사용자 페르소나

---

## Persona 1: 슈퍼 관리자 (Super Admin)

| 항목 | 내용 |
|---|---|
| **이름** | 김본사 |
| **역할** | 본사 시스템 관리자 |
| **나이** | 35세 |
| **기술 수준** | 중상 (웹 기반 관리 도구 사용에 익숙) |

**배경**: 프랜차이즈 본사에서 여러 매장을 총괄 관리하는 담당자. 매장 등록, 관리자 계정 생성, 광고 이미지 관리 등 시스템 전반의 설정을 담당한다.

**목표**:
- 새로운 매장을 빠르게 시스템에 등록하고 운영 준비를 완료
- 각 매장별 관리자 계정을 효율적으로 관리
- 매장별 광고 이미지를 등록하여 고객 경험을 향상

**불만 사항 (Pain Points)**:
- 매장이 늘어날수록 개별 설정 작업이 번거로움
- 광고 이미지 교체 시 매장별로 일일이 작업해야 하는 비효율

**사용 빈도**: 주 2~3회 (매장 추가, 광고 업데이트 시)

## Persona 2: 매장 관리자 (Store Admin)

| 항목 | 내용 |
|---|---|
| **이름** | 이매장 |
| **역할** | 매장 운영 관리자 (점주 또는 매니저) |
| **나이** | 42세 |
| **기술 수준** | 중 (스마트폰, 기본 PC 사용 가능) |

**배경**: 개별 매장을 운영하는 점주 또는 매니저. 매일 매장을 열고 닫으며, 주문을 실시간으로 모니터링하고 테이블을 관리한다. 메뉴 변경이 있을 때 직접 시스템에 반영한다.

**목표**:
- 들어오는 주문을 실시간으로 빠르게 확인하고 처리
- 테이블 회전율을 높이기 위해 효율적으로 세션을 관리
- 메뉴를 쉽게 등록/수정하여 최신 상태 유지

**불만 사항 (Pain Points)**:
- 바쁜 시간대에 주문을 놓치는 것에 대한 불안
- 테이블 이용 완료 처리를 잊어서 다음 고객에게 이전 주문이 보이는 실수
- 메뉴 이미지 등록이 복잡하면 업데이트를 미루게 됨

**사용 빈도**: 매일 (영업시간 내내)

## Persona 3: 고객 (Customer / Table User)

| 항목 | 내용 |
|---|---|
| **이름** | 박고객 |
| **역할** | 매장 방문 고객 |
| **나이** | 28세 |
| **기술 수준** | 다양 (스마트폰 사용에 익숙한 일반인) |

**배경**: 매장에 방문하여 테이블에 앉은 고객. 태블릿을 통해 메뉴를 보고 주문한다. 직원을 부르지 않고 편하게 주문하고 싶어한다. 친구들과 함께 올 때 누가 계산할지 재미있게 정하고 싶어한다.

**목표**:
- 빠르고 직관적으로 메뉴를 탐색하고 주문
- 주문 상태를 실시간으로 확인
- 친구들과 함께 식사할 때 사다리 타기로 재미있게 결제자 결정

**불만 사항 (Pain Points)**:
- 복잡한 UI로 인해 주문에 시간이 오래 걸림
- 주문이 제대로 들어갔는지 확인이 안 됨
- 이전 손님의 주문 내역이 보이면 불쾌함

**사용 빈도**: 매장 방문 시 (1회성, 세션 기반)
