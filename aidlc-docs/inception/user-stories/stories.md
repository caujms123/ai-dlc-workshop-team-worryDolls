# 테이블오더 서비스 - User Stories

> **분류 방식**: 사용자 유형별 (Persona-Based)
> **상세 수준**: 표준 (스토리당 3~5개 수용 기준)
> **INVEST 기준 준수**: Independent, Negotiable, Valuable, Estimable, Small, Testable

---

# 슈퍼 관리자 (김본사) 스토리

## Epic: 매장 관리

### US-SA-01: 매장 등록
**As a** 슈퍼 관리자,
**I want to** 새로운 매장을 시스템에 등록할 수 있기를,
**So that** 해당 매장에서 테이블오더 서비스를 사용할 수 있다.

**Acceptance Criteria**:
1. 매장명, 매장 식별자, 주소를 입력하여 매장을 등록할 수 있다
2. 매장 식별자는 시스템 내에서 고유해야 한다
3. 필수 필드(매장명, 매장 식별자)가 비어있으면 등록이 거부되고 에러 메시지가 표시된다
4. 등록 성공 시 매장 목록에 새 매장이 즉시 표시된다

**의존성**: 없음

---

### US-SA-02: 매장 정보 수정
**As a** 슈퍼 관리자,
**I want to** 등록된 매장의 정보를 수정할 수 있기를,
**So that** 매장 정보를 최신 상태로 유지할 수 있다.

**Acceptance Criteria**:
1. 매장 목록에서 특정 매장을 선택하여 정보를 수정할 수 있다
2. 매장명, 주소 등의 정보를 변경할 수 있다
3. 수정 성공 시 변경된 정보가 즉시 반영된다
4. 필수 필드 검증이 수정 시에도 동일하게 적용된다

**의존성**: US-SA-01

---

### US-SA-03: 매장 목록 조회
**As a** 슈퍼 관리자,
**I want to** 등록된 모든 매장의 목록을 조회할 수 있기를,
**So that** 전체 매장 현황을 파악할 수 있다.

**Acceptance Criteria**:
1. 등록된 모든 매장이 목록 형태로 표시된다
2. 매장명, 매장 식별자, 주소가 목록에 표시된다
3. 매장을 클릭하면 상세 정보를 확인할 수 있다

**의존성**: US-SA-01

---

### US-SA-04: 매장별 관리자 계정 관리
**As a** 슈퍼 관리자,
**I want to** 각 매장에 관리자 계정을 생성하고 관리할 수 있기를,
**So that** 매장 운영자가 시스템에 접근할 수 있다.

**Acceptance Criteria**:
1. 특정 매장에 대해 관리자 계정(사용자명, 비밀번호)을 생성할 수 있다
2. 매장별 관리자 계정 목록을 조회할 수 있다
3. 관리자 계정을 비활성화할 수 있다
4. 동일 매장 내 사용자명 중복은 허용되지 않는다
5. 비밀번호는 bcrypt로 해싱되어 저장된다

**의존성**: US-SA-01

---

## Epic: 광고 관리

### US-SA-05: 광고 이미지 업로드
**As a** 슈퍼 관리자,
**I want to** 매장별 광고 이미지를 업로드할 수 있기를,
**So that** 고객 태블릿에 광고가 표시된다.

**Acceptance Criteria**:
1. 특정 매장을 선택하여 광고 이미지 파일을 업로드할 수 있다
2. 이미지 파일이 서버에 저장되고 URL이 생성된다
3. 업로드 성공 시 광고 이미지 목록에 즉시 표시된다
4. 지원 이미지 형식(JPG, PNG)이 아닌 파일은 업로드가 거부된다

**의존성**: US-SA-01

---

### US-SA-06: 광고 이미지 관리
**As a** 슈퍼 관리자,
**I want to** 등록된 광고 이미지의 순서, 활성 상태를 관리하고 삭제할 수 있기를,
**So that** 고객에게 보여줄 광고를 효과적으로 운영할 수 있다.

**Acceptance Criteria**:
1. 광고 이미지 목록을 조회할 수 있다
2. 광고 이미지의 노출 순서를 드래그 또는 버튼으로 조정할 수 있다
3. 광고 이미지를 활성/비활성 상태로 전환할 수 있다
4. 광고 이미지를 삭제할 수 있다 (확인 팝업 표시)
5. 비활성 광고는 고객 태블릿에 표시되지 않는다

**의존성**: US-SA-05

---

## Epic: 슈퍼 관리자 인증

### US-SA-07: 슈퍼 관리자 로그인
**As a** 슈퍼 관리자,
**I want to** 사용자명과 비밀번호로 시스템에 로그인할 수 있기를,
**So that** 시스템 관리 기능에 접근할 수 있다.

**Acceptance Criteria**:
1. 사용자명과 비밀번호를 입력하여 로그인할 수 있다
2. 로그인 성공 시 JWT 토큰이 발급되고 관리자 대시보드로 이동한다
3. 잘못된 자격 증명 시 에러 메시지가 표시된다
4. 연속 로그인 실패 시 일정 시간 로그인이 제한된다
5. 브라우저 새로고침 시 세션이 유지된다

**의존성**: 없음

# 매장 관리자 (이매장) 스토리

## Epic: 매장 관리자 인증

### US-MA-01: 매장 관리자 로그인
**As a** 매장 관리자,
**I want to** 매장 식별자, 사용자명, 비밀번호로 로그인할 수 있기를,
**So that** 내 매장의 관리 기능에 접근할 수 있다.

**Acceptance Criteria**:
1. 매장 식별자, 사용자명, 비밀번호를 입력하여 로그인할 수 있다
2. 로그인 성공 시 JWT 토큰이 발급되고 16시간 세션이 생성된다
3. 잘못된 자격 증명 시 에러 메시지가 표시된다
4. 연속 로그인 실패 시 일정 시간 로그인이 제한된다
5. 16시간 후 자동 로그아웃되며 재로그인이 필요하다

**의존성**: US-SA-04

---

## Epic: 메뉴 관리

### US-MA-02: 메뉴 조회
**As a** 매장 관리자,
**I want to** 내 매장의 메뉴를 카테고리별로 조회할 수 있기를,
**So that** 현재 등록된 메뉴 현황을 파악할 수 있다.

**Acceptance Criteria**:
1. 카테고리별로 메뉴 목록이 표시된다
2. 각 메뉴의 메뉴명, 가격, 설명, 이미지, 노출 순서가 표시된다
3. 내 매장의 메뉴만 표시된다 (다른 매장 메뉴 접근 불가)

**의존성**: US-MA-01

---

### US-MA-03: 메뉴 등록
**As a** 매장 관리자,
**I want to** 새로운 메뉴를 등록할 수 있기를,
**So that** 고객이 새 메뉴를 주문할 수 있다.

**Acceptance Criteria**:
1. 메뉴명, 가격, 설명, 카테고리를 입력하여 메뉴를 등록할 수 있다
2. 메뉴 이미지 파일을 업로드할 수 있다
3. 필수 필드(메뉴명, 가격, 카테고리)가 비어있으면 등록이 거부된다
4. 가격은 0 이상이어야 한다
5. 등록 성공 시 메뉴 목록에 즉시 표시된다

**의존성**: US-MA-01

---

### US-MA-04: 메뉴 수정
**As a** 매장 관리자,
**I want to** 기존 메뉴의 정보를 수정할 수 있기를,
**So that** 메뉴 정보를 최신 상태로 유지할 수 있다.

**Acceptance Criteria**:
1. 메뉴 목록에서 특정 메뉴를 선택하여 정보를 수정할 수 있다
2. 메뉴명, 가격, 설명, 카테고리, 이미지를 변경할 수 있다
3. 수정 시에도 필수 필드 검증과 가격 범위 검증이 적용된다
4. 수정 성공 시 변경 사항이 즉시 반영된다

**의존성**: US-MA-03

---

### US-MA-05: 메뉴 삭제
**As a** 매장 관리자,
**I want to** 더 이상 판매하지 않는 메뉴를 삭제할 수 있기를,
**So that** 고객에게 정확한 메뉴만 표시된다.

**Acceptance Criteria**:
1. 메뉴 목록에서 특정 메뉴를 삭제할 수 있다
2. 삭제 전 확인 팝업이 표시된다
3. 삭제 성공 시 메뉴 목록에서 즉시 제거된다
4. 삭제된 메뉴는 고객 화면에 더 이상 표시되지 않는다

**의존성**: US-MA-03

---

### US-MA-06: 메뉴 노출 순서 조정
**As a** 매장 관리자,
**I want to** 메뉴의 노출 순서를 조정할 수 있기를,
**So that** 인기 메뉴나 추천 메뉴를 상단에 배치할 수 있다.

**Acceptance Criteria**:
1. 카테고리 내에서 메뉴의 노출 순서를 변경할 수 있다
2. 드래그 앤 드롭 또는 순서 변경 버튼으로 조정 가능하다
3. 변경된 순서가 고객 화면에 즉시 반영된다

**의존성**: US-MA-03

---

## Epic: 실시간 주문 모니터링

### US-MA-07: 실시간 주문 대시보드 조회
**As a** 매장 관리자,
**I want to** 테이블별 주문 현황을 실시간 대시보드로 확인할 수 있기를,
**So that** 들어오는 주문을 빠르게 파악하고 처리할 수 있다.

**Acceptance Criteria**:
1. 테이블별 카드 형태의 그리드 레이아웃으로 주문 현황이 표시된다
2. 각 테이블 카드에 총 주문액과 최신 주문 미리보기가 표시된다
3. SSE를 통해 신규 주문이 2초 이내에 화면에 반영된다
4. 신규 주문은 색상 변경 또는 애니메이션으로 시각적으로 강조된다
5. 테이블별 필터링이 가능하다

**의존성**: US-MA-01

---

### US-MA-08: 주문 상세 조회
**As a** 매장 관리자,
**I want to** 특정 테이블의 주문 상세 내역을 확인할 수 있기를,
**So that** 주문 내용을 정확히 파악할 수 있다.

**Acceptance Criteria**:
1. 테이블 카드를 클릭하면 해당 테이블의 전체 주문 목록이 표시된다
2. 각 주문의 주문 번호, 시각, 메뉴 목록, 수량, 총 금액이 표시된다
3. 주문 상태(대기중/준비중/완료)가 표시된다

**의존성**: US-MA-07

---

### US-MA-09: 주문 상태 변경
**As a** 매장 관리자,
**I want to** 주문의 상태를 변경할 수 있기를,
**So that** 주문 처리 진행 상황을 추적하고 고객에게 알릴 수 있다.

**Acceptance Criteria**:
1. 주문 상태를 대기중 → 준비중 → 완료로 변경할 수 있다
2. 상태 변경 시 대시보드에 즉시 반영된다
3. 상태 변경이 SSE를 통해 고객 화면에도 실시간 반영된다
4. 상태 변경 성공/실패 피드백이 표시된다

**의존성**: US-MA-08

---

## Epic: 테이블 관리

### US-MA-10: 테이블 초기 설정
**As a** 매장 관리자,
**I want to** 테이블 태블릿의 초기 설정(테이블 번호, 비밀번호)을 할 수 있기를,
**So that** 고객이 해당 테이블에서 주문할 수 있다.

**Acceptance Criteria**:
1. 테이블 번호와 비밀번호를 설정할 수 있다
2. 설정 완료 시 16시간 세션이 생성된다
3. 설정 정보가 저장되어 태블릿에서 자동 로그인이 활성화된다
4. 성공/실패 피드백이 표시된다

**의존성**: US-MA-01

---

### US-MA-11: 주문 삭제 (직권 수정)
**As a** 매장 관리자,
**I want to** 잘못된 주문을 삭제할 수 있기를,
**So that** 주문 오류를 즉시 수정할 수 있다.

**Acceptance Criteria**:
1. 특정 주문에 대해 삭제 버튼을 클릭할 수 있다
2. 삭제 전 확인 팝업이 표시된다
3. 삭제 시 주문이 즉시 제거되고 테이블 총 주문액이 재계산된다
4. 삭제 성공/실패 피드백이 표시된다

**의존성**: US-MA-08

---

### US-MA-12: 테이블 이용 완료 처리
**As a** 매장 관리자,
**I want to** 고객이 떠난 테이블의 이용 완료 처리를 할 수 있기를,
**So that** 다음 고객이 깨끗한 상태에서 시작할 수 있다.

**Acceptance Criteria**:
1. 테이블 카드에서 "이용 완료" 버튼을 클릭할 수 있다
2. 확인 팝업이 표시된다
3. 이용 완료 시 해당 세션의 주문 내역이 과거 이력으로 이동된다
4. 테이블의 현재 주문 목록과 총 주문액이 0으로 리셋된다
5. 성공/실패 피드백이 표시된다

**의존성**: US-MA-07

---

### US-MA-13: 과거 주문 내역 조회
**As a** 매장 관리자,
**I want to** 테이블의 과거 주문 내역을 조회할 수 있기를,
**So that** 이전 세션의 주문 이력을 확인할 수 있다.

**Acceptance Criteria**:
1. "과거 내역" 버튼을 클릭하면 과거 주문 목록이 표시된다
2. 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
3. 시간 역순으로 정렬된다
4. 날짜 필터링이 가능하다
5. "닫기" 버튼으로 대시보드로 복귀할 수 있다

**의존성**: US-MA-12

# 고객 (박고객) 스토리

## Epic: 테이블 인증

### US-CU-01: 테이블 태블릿 자동 로그인
**As a** 고객,
**I want to** 테이블에 앉으면 별도 로그인 없이 바로 서비스를 이용할 수 있기를,
**So that** 대기 시간 없이 즉시 주문할 수 있다.

**Acceptance Criteria**:
1. 태블릿에 저장된 정보로 자동 로그인이 수행된다
2. 자동 로그인 성공 시 광고 화면(기본 화면)이 표시된다
3. 자동 로그인 실패 시 초기 설정 화면이 표시된다
4. 16시간 세션이 유지되며, 만료 시 자동 재로그인을 시도한다

**의존성**: US-MA-10

---

## Epic: 광고 화면

### US-CU-02: 광고 화면 표시
**As a** 고객,
**I want to** 태블릿에서 매장의 광고를 볼 수 있기를,
**So that** 매장의 프로모션이나 이벤트 정보를 확인할 수 있다.

**Acceptance Criteria**:
1. 로그인 후 광고 화면이 기본 화면(랜딩 페이지)으로 표시된다
2. 등록된 광고 이미지가 5초 간격으로 자동 슬라이드된다
3. 활성 상태의 광고 이미지만 표시된다
4. 노출 순서에 따라 이미지가 순서대로 표시된다
5. 등록된 광고가 없을 경우 기본 이미지 또는 바로 메뉴 화면이 표시된다

**의존성**: US-SA-05, US-CU-01

### US-CU-03: 광고 화면에서 메뉴 화면으로 이동
**As a** 고객,
**I want to** 광고 화면을 터치하면 메뉴 화면으로 이동할 수 있기를,
**So that** 원할 때 바로 주문을 시작할 수 있다.

**Acceptance Criteria**:
1. 광고 화면의 아무 곳이나 한 번 터치하면 메뉴 화면으로 이동한다
2. 메뉴 화면으로의 전환이 즉시(1초 이내) 이루어진다
3. 일정 시간(예: 2분) 동안 조작이 없으면 메뉴 화면에서 광고 화면으로 자동 복귀한다

**의존성**: US-CU-02

### US-CU-04: 카테고리별 메뉴 조회

4. 터치 타겟이 최소 44x44px 이상이다
5. 메뉴 카드를 터치하면 상세 정보(설명 포함)가 표시된다

**의존성**: US-CU-03

## Epic: 장바구니

### US-CU-05: 장바구니에 메뉴 추가
**As a** 고객,
**I want to** 원하는 메뉴를 장바구니에 추가할 수 있기를,
**So that** 여러 메뉴를 한 번에 주문할 수 있다.

1. 메뉴 카드 또는 상세 화면에서 장바구니에 추가할 수 있다
2. 추가 시 장바구니 아이콘에 수량이 표시된다
3. 동일 메뉴를 다시 추가하면 수량이 증가한다
4. 장바구니 데이터가 로컬에 저장되어 페이지 새로고침 시에도 유지된다

**의존성**: US-CU-04

### US-CU-06: 장바구니 관리
**As a** 고객,
**I want to** 장바구니의 메뉴 수량을 조절하거나 삭제할 수 있기를,
**So that** 주문 전에 원하는 대로 수정할 수 있다.

**Acceptance Criteria**:
1. 장바구니에서 각 메뉴의 수량을 증가/감소할 수 있다
2. 수량이 0이 되면 해당 메뉴가 장바구니에서 제거된다
3. 총 금액이 수량 변경 시 실시간으로 재계산된다
4. "장바구니 비우기" 버튼으로 전체 항목을 삭제할 수 있다
5. 서버 전송은 주문 확정 시에만 수행된다

**의존성**: US-CU-05

## Epic: 주문 생성

### US-CU-07: 결제 방식 선택
**As a** 고객,
**I want to** 주문 확정 전에 더치페이 또는 단독 계산을 선택할 수 있기를,
**So that** 원하는 결제 방식으로 주문할 수 있다.

**Acceptance Criteria**:
1. 장바구니에서 "주문하기"를 누르면 결제 방식 선택 팝업이 먼저 표시된다
2. "더치페이 (1/N 균등 분할)" 옵션을 선택할 수 있다
3. "단독 계산 (한 명이 전액)" 옵션을 선택할 수 있다
4. 결제 방식 선택 후 주문 내역 최종 확인 화면으로 이동한다
5. 팝업 오른쪽 상단에 사다리 타기 버튼이 표시된다

**의존성**: US-CU-06

### US-CU-08: 사다리 타기 미니게임
**As a** 고객,
**I want to** 사다리 타기 게임으로 결제자를 재미있게 정할 수 있기를,
**So that** 친구들과 함께 식사할 때 즐거운 경험을 할 수 있다.

**Acceptance Criteria**:
1. 결제 방식 선택 팝업의 오른쪽 상단 버튼을 누르면 인원 선택 화면으로 이동한다
2. 인원 수(2~10명)를 선택할 수 있다
3. 인원 선택 후 사다리 타기 게임 화면이 표시된다
4. "시작" 버튼을 누르면 사다리 형태 UI에서 부드러운 이동 애니메이션이 실행된다
5. 꽝(결제자)이 결정되면 결과가 강조 연출과 함께 표시된다
6. 결과 표시 후 5초 대기 후 결제 방식 선택 팝업으로 자동 복귀한다 (결제자 표시 없음)

**의존성**: US-CU-07

### US-CU-09: 주문 확정
**As a** 고객,
**I want to** 주문 내역을 최종 확인하고 주문을 확정할 수 있기를,
**So that** 정확한 주문이 매장에 전달된다.

**Acceptance Criteria**:
1. 주문 내역(메뉴명, 수량, 단가, 총 금액)이 최종 확인 화면에 표시된다
2. 선택한 결제 방식(더치페이/단독)이 표시된다
3. "주문 확정" 버튼을 누르면 주문이 서버로 전송된다
4. 주문 성공 시 주문 번호가 표시되고, 장바구니가 자동으로 비워지며, 5초 후 메뉴 화면으로 자동 리다이렉트된다
5. 주문 실패 시 에러 메시지가 표시되고 장바구니가 유지된다

**의존성**: US-CU-07
