# Unit 4: 비즈니스 규칙 - 고객 UI + 테이블 관리

---

## 1. 테이블 규칙

### BR-TABLE-01: 테이블 등록
- 매장 관리자만 등록 가능
- 동일 매장 내 table_number 중복 불가
- 비밀번호는 bcrypt 해싱 저장
- 비밀번호 최소 4자 (태블릿 간편 비밀번호)

### BR-TABLE-02: 테이블 세션 라이프사이클
1. **세션 시작**: 해당 테이블의 첫 주문 생성 시 자동 시작
2. **세션 활성**: 주문 생성/조회 가능
3. **세션 종료**: 관리자가 "이용 완료" 처리 시
   - 해당 세션의 주문을 OrderHistory로 이동 (Unit 3)
   - 세션 is_active=false, ended_at 기록
   - 테이블 현재 주문 목록/총액 리셋

### BR-TABLE-03: 이용 완료 처리
- 매장 관리자만 처리 가능
- 확인 팝업 필수
- 처리 순서: 주문 이력 이동 → 세션 종료 → SSE 이벤트 발행
- 새 고객의 첫 주문 시 새 세션 자동 시작

## 2. 고객 UI 규칙

### BR-CUI-01: 광고 화면
- 로그인 후 기본 화면 (랜딩 페이지)
- 활성 광고 이미지 5초 간격 자동 슬라이드
- 화면 아무 곳이나 터치 시 메뉴 화면으로 이동
- 2분간 조작 없으면 메뉴 화면에서 광고 화면으로 자동 복귀
- 광고가 없으면 기본 이미지 또는 바로 메뉴 화면

### BR-CUI-02: 장바구니
- 클라이언트 로컬 스토리지에 저장 (localStorage)
- 페이지 새로고침 시에도 유지
- 저장 형식: `{ storeId, items: [{menuId, menuName, price, quantity, imageUrl}] }`
- 수량 변경 시 총 금액 실시간 재계산
- 수량 0이면 항목 자동 제거
- 주문 성공 시 장바구니 자동 비우기

### BR-CUI-03: 결제 방식 선택
- 장바구니에서 "주문하기" 클릭 시 팝업 표시
- 옵션: 더치페이 (1/N 균등 분할) / 단독 계산
- 오른쪽 상단에 사다리 타기 버튼
- 결제 방식 선택 후 주문 확정 화면으로 이동

### BR-CUI-04: 사다리 타기 미니게임
- 인원 수 선택: 2~10명
- 사다리 형태 UI 생성 (인원 수에 따라 세로줄 수 결정)
- 가로줄은 랜덤 생성 (각 구간에 0~2개)
- "시작" 버튼 클릭 시 부드러운 이동 애니메이션 실행
- 꽝(결제자) 1명 결정
- 결과 강조 연출 (색상 변경, 크기 확대, 효과음)
- 결과 표시 후 5초 대기
- 5초 후 결제 방식 선택 팝업으로 자동 복귀 (결제자 표시 없음)
- 사다리 로직은 순수 클라이언트 사이드 (서버 통신 없음)

### BR-CUI-05: 자동 로그인
- 로컬 스토리지에 저장된 토큰 확인
- 토큰 유효 → 광고 화면으로 이동
- 토큰 만료 → 저장된 자격 증명으로 재로그인 시도
- 재로그인 실패 → 초기 설정 화면 표시
