## 4. 기능 요구사항 (Functional Requirements)

### FR-01: 매장 관리 (슈퍼 관리자)
- **FR-01-01**: 매장 등록 (매장명, 매장 식별자, 주소 등)
- **FR-01-02**: 매장 정보 수정
- **FR-01-03**: 매장 목록 조회
- **FR-01-04**: 매장별 관리자 계정 생성/관리

### FR-01-05: 광고 이미지 관리 (슈퍼 관리자)
- **FR-01-05-01**: 매장별 광고 이미지 업로드 (서버에 이미지 파일 저장)
- **FR-01-05-02**: 광고 이미지 목록 조회
- **FR-01-05-03**: 광고 이미지 삭제
- **FR-01-05-04**: 광고 이미지 노출 순서 조정
- **FR-01-05-05**: 광고 이미지 활성/비활성 설정
- **FR-01-05-06**: 고객 태블릿 광고 화면에서 등록된 광고 이미지 표시 (여러 장일 경우 슬라이드 형태)

### FR-02: 인증 및 세션 관리
- **FR-02-01**: 슈퍼 관리자 로그인 (사용자명/비밀번호, JWT)
- **FR-02-02**: 매장 관리자 로그인 (매장 식별자 + 사용자명/비밀번호, JWT, 16시간 세션)
- **FR-02-03**: 테이블 태블릿 초기 설정 (매장 식별자, 테이블 번호, 비밀번호)
- **FR-02-04**: 테이블 태블릿 자동 로그인 (저장된 정보 기반, 16시간 세션)
- **FR-02-05**: 비밀번호 bcrypt 해싱
- **FR-02-06**: 로그인 시도 제한
- **FR-02-07**: 브라우저 새로고침 시 세션 유지
- **FR-02-08**: 세션 만료 시 자동 로그아웃

### FR-03: 메뉴 관리 (관리자)
- **FR-03-01**: 카테고리별 메뉴 조회
- **FR-03-02**: 메뉴 등록 (메뉴명, 가격, 설명, 카테고리, 이미지)
- **FR-03-03**: 메뉴 수정
- **FR-03-04**: 메뉴 삭제
- **FR-03-05**: 메뉴 노출 순서 조정
- **FR-03-06**: 이미지 파일 서버 업로드
- **FR-03-07**: 필수 필드 검증 (메뉴명, 가격, 카테고리)
- **FR-03-08**: 가격 범위 검증 (0 이상)

### FR-04: 광고 화면 및 메뉴 조회 (고객)
- **FR-04-01**: 광고 화면이 기본 화면(랜딩 페이지)으로 표시
- **FR-04-02**: 광고 화면을 사용자가 한 번 터치하면 메뉴 화면으로 이동
- **FR-04-03**: 카테고리별 메뉴 분류 및 표시
- **FR-04-04**: 메뉴 상세 정보 표시 (메뉴명, 가격, 설명, 이미지)
- **FR-04-05**: 카테고리 간 빠른 이동
- **FR-04-06**: 카드 형태 메뉴 레이아웃
- **FR-04-07**: 터치 친화적 버튼 (최소 44x44px)

### FR-05: 장바구니 관리 (고객)
- **FR-05-01**: 메뉴 추가/삭제
- **FR-05-02**: 수량 조절 (증가/감소)
- **FR-05-03**: 총 금액 실시간 계산
- **FR-05-04**: 장바구니 비우기
- **FR-05-05**: 클라이언트 측 로컬 저장 (페이지 새로고침 시 유지)
- **FR-05-06**: 서버 전송은 주문 확정 시에만 수행

### FR-06: 주문 생성 (고객)
- **FR-06-01**: 주문 내역 최종 확인 화면
- **FR-06-02**: 결제 방식 선택 팝업 (주문 확정 전 표시)
  - 더치페이 (1/N 균등 분할) 선택
  - 단독 계산 (한 명이 전액 결제) 선택
- **FR-06-03**: 사다리 타기 미니게임 (결제자 결정)
  - 결제 방식 선택 팝업 오른쪽 상단에 사다리 타기 버튼 배치
  - 버튼 클릭 시 인원 수 선택 화면으로 이동
  - 인원 선택 후 사다리 타기 게임 화면으로 이동
  - "시작" 버튼 클릭 시 사다리 애니메이션 실행
  - 꽝(결제자) 결정 후 결과 표시
  - 결과 표시 후 5초 대기, 이후 결제 단계로 자동 이동
- **FR-06-04**: 주문 확정 버튼
- **FR-06-05**: 주문 성공 시: 주문 번호 표시, 장바구니 자동 비우기, 5초 후 메뉴 화면 자동 리다이렉트
- **FR-06-06**: 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- **FR-06-07**: 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID, 결제 방식(더치페이/단독)

### FR-07: 주문 내역 조회 (고객)
- **FR-07-01**: 주문 시간 순 정렬
- **FR-07-02**: 주문별 상세 정보 (주문 번호, 시각, 메뉴/수량, 금액, 상태)
- **FR-07-03**: 주문 상태 실시간 업데이트 (SSE)
- **FR-07-04**: 현재 테이블 세션 주문만 표시
- **FR-07-05**: 매장 이용 완료 처리된 주문 제외
- **FR-07-06**: 페이지네이션 또는 무한 스크롤

### FR-08: 실시간 주문 모니터링 (관리자)
- **FR-08-01**: SSE 기반 주문 목록 실시간 업데이트
- **FR-08-02**: 테이블별 카드 형태 그리드/대시보드 레이아웃
- **FR-08-03**: 각 테이블 카드에 총 주문액 표시
- **FR-08-04**: 최신 주문 n개 미리보기
- **FR-08-05**: 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- **FR-08-06**: 주문 상태 변경 (대기중/준비중/완료)
- **FR-08-07**: 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- **FR-08-08**: 2초 이내 주문 표시
- **FR-08-09**: 테이블별 필터링 기능

### FR-09: 테이블 관리 (관리자)
- **FR-09-01**: 테이블 번호 및 비밀번호 설정
- **FR-09-02**: 16시간 세션 생성
- **FR-09-03**: 주문 삭제 (확인 팝업, 즉시 삭제, 총 주문액 재계산)
- **FR-09-04**: 테이블 세션 종료 (이용 완료) - 확인 팝업, 주문 내역 과거 이력 이동, 주문 목록/총액 리셋
- **FR-09-05**: 과거 주문 내역 조회 (시간 역순, 날짜 필터링)
- **FR-09-06**: 성공/실패 피드백

### FR-10: 테이블 세션 라이프사이클
- **FR-10-01**: 세션 시작: 해당 테이블의 첫 주문 시 자동 시작
- **FR-10-02**: 세션 종료: 관리자가 "이용 완료" 처리 시
- **FR-10-03**: 세션 종료 시 주문 내역 OrderHistory로 이동
- **FR-10-04**: 새 고객은 이전 주문 내역 없이 시작

## 5. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- **NFR-01-01**: SSE 기반 실시간 주문 알림 2초 이내 전달
- **NFR-01-02**: 메뉴 조회 API 응답 시간 500ms 이내
- **NFR-01-03**: 주문 생성 API 응답 시간 1초 이내
- **NFR-01-04**: 동시 50~100 테이블 접속 지원

### NFR-02: 보안
- **NFR-02-01**: JWT 토큰 기반 인증
- **NFR-02-02**: 비밀번호 bcrypt 해싱
- **NFR-02-03**: 로그인 시도 제한 (brute-force 방지)
- **NFR-02-04**: 역할 기반 접근 제어 (슈퍼 관리자, 매장 관리자, 테이블)
- **NFR-02-05**: API 입력값 검증
- **NFR-02-06**: SQL Injection 방지 (파라미터화된 쿼리)

### NFR-03: 가용성
- **NFR-03-01**: 매장 영업시간 동안 서비스 가용성 99% 이상
- **NFR-03-02**: 서버 장애 시 적절한 에러 메시지 표시

### NFR-04: 사용성
- **NFR-04-01**: 터치 친화적 UI (최소 44x44px 터치 타겟)
- **NFR-04-02**: 직관적인 네비게이션
- **NFR-04-03**: 반응형 레이아웃 (태블릿 최적화)
- **NFR-04-04**: 명확한 시각적 피드백 (로딩, 성공, 에러)

### NFR-05: 유지보수성
- **NFR-05-01**: 계층형 아키텍처 (Router - Service - Repository)
- **NFR-05-02**: RESTful API 설계
- **NFR-05-03**: 코드 문서화 및 주석
- **NFR-05-04**: FastAPI 자동 API 문서 (Swagger/OpenAPI) 활용

### NFR-06: 데이터 무결성
- **NFR-06-01**: 주문 데이터 트랜잭션 처리
- **NFR-06-02**: 매장별 데이터 격리
- **NFR-06-03**: 주문 이력 영구 보존

## 6. 제외 기능 (Out of Scope)

requirements/constraints.md에 정의된 대로 다음 기능은 구현하지 않음:
- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 리사이징/최적화, CMS
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 재고, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도, 번역)

## 7. MVP 범위 정리

### 고객용 (Customer)
- 테이블 태블릿 자동 로그인 및 세션 관리 (16시간)
- 광고 화면 (기본 랜딩 페이지, 터치 시 메뉴 화면 이동)
- 메뉴 조회 (카테고리별, 카드 레이아웃)
- 장바구니 관리 (로컬 저장)
- 주문 생성 (결제 방식 선택: 더치페이/단독 계산, 사다리 타기 미니게임, 5초 표시 후 자동 리다이렉트)
- 주문 내역 조회 (현재 세션, SSE 실시간 상태 업데이트)

### 관리자용 (Admin)
- 슈퍼 관리자: 매장 관리, 매장별 관리자 계정 관리, 광고 이미지 관리
- 매장 관리자: 매장 인증 (16시간 세션)
- 실시간 주문 모니터링 (그리드 레이아웃, SSE)
- 테이블 관리 (초기 설정, 주문 삭제, 이용 완료, 과거 내역 조회)
- 메뉴 관리 (CRUD 전체, 이미지 업로드, 순서 조정)

## 8. 데이터 모델 개요

### 주요 엔티티
- **Advertisement**: 광고 이미지 (매장 연결, 이미지 경로, 노출 순서, 활성 여부)
- **Store**: 매장 정보 (매장명, 식별자, 주소 등)
- **Admin**: 관리자 계정 (슈퍼 관리자, 매장 관리자)
- **TableInfo**: 테이블 정보 (테이블 번호, 비밀번호, 매장 연결)
- **TableSession**: 테이블 세션 (시작 시각, 종료 시각, 세션 ID)
- **Category**: 메뉴 카테고리
- **Menu**: 메뉴 항목 (메뉴명, 가격, 설명, 이미지, 카테고리, 노출 순서)
- **Order**: 주문 (주문 번호, 테이블, 세션, 상태, 총 금액, 주문 시각)
- **OrderItem**: 주문 항목 (메뉴, 수량, 단가)
- **OrderHistory**: 과거 주문 이력 (세션 종료 시 이동)

## 9. Security Extension 결정

사용자가 "규모에 맞게 알아서 해달라"고 요청하였으므로, 프로젝트 특성을 고려하여 다음과 같이 결정:

- **적용**: 중규모 프로덕션 서비스이므로 Security Extension **적용**
- **적용 범위 조정**:
  - 애플리케이션 레벨 보안 규칙 (SECURITY-03, 04, 05, 08, 09, 10, 11, 12, 13, 15): **적용**
  - 인프라/클라우드 관련 규칙 (SECURITY-01, 02, 06, 07, 14): 온프레미스 환경이므로 **N/A** (해당 없음으로 처리, 향후 클라우드 이전 시 적용)
